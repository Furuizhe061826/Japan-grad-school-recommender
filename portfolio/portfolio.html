User Problem & Product Flow

用户痛点与产品流程

用户痛点

  • 日本大学院申请信息分散，学生难以快速判断院校、研究科和教授方向是否匹配。
  • 学生背景差异明显，传统选校建议常依赖经验判断，缺乏结构化和可复用的决策逻辑。
  • 募集要项、研究室、教授方向和语言要求交叉复杂，信息检索和咨询成本较高。

产品目标

  • 将用户背景转化为结构化字段，降低选校判断门槛。
  • 基于本地院校与教授数据，形成初步候选池。
  • 通过规则打分输出可解释推荐，辅助用户制定申请策略。
01 输入背景 本科院校、专业、GPA、语言成绩、研究方向、地区偏好
02 信息结构化 识别院校层次、语言门槛、研究关键词和目标学位
03 数据库匹配 匹配院校、研究科、方向关键词和潜在教授资料
04 推荐打分 综合研究匹配、申请难度、语言门槛和地区偏好
05 输出报告 生成冲刺、匹配、相对稳妥三档推荐和提升建议
设计重点：不是简单展示院校列表，而是把申请咨询过程拆解为可填写字段、可维护数据、可解释规则和可复用报告。

Recommendation Logic & Data Structure

推荐逻辑与数据结构

数据结构

  • 本地 JSON 数据库存储大学、研究科、专业方向、申请难度、语言要求和关键词。
  • 教授资料库存储学校、所属组织、职位、研究关键词、研究摘要和官方链接。
  • 中国本科院校识别库用于辅助判断本科背景层次，提升输入字段的结构化程度。

推荐输出

  • 冲刺 / 匹配 / 相对稳妥三档推荐结果。
  • 每个项目提供匹配分数、推荐理由、潜在导师和提升建议。
  • 报告语言尽量接近咨询建议，而非单纯表格排序。

Value & Iteration Plan

项目价值与后续迭代

项目价值

  • 将真实留学咨询需求转化为可交互产品 MVP，体现需求理解与产品抽象能力。
  • 完成从需求拆解、字段设计、推荐规则、页面展示到部署上线的完整闭环。
  • 通过本地数据结构和规则算法模拟 AI 推荐过程，体现数据产品设计能力。
  • 使用 AI 辅助开发持续迭代功能，体现快速学习、问题拆解和执行落地能力。

能力映射

  • 项目管理：需求拆解、优先级判断、版本迭代和上线部署。
  • 数据分析：字段建模、规则打分、数据质量校验和结果解释。
  • AI 产品：推荐逻辑设计、可解释输出、未来 RAG / LLM 接入规划。
  • 全球化业务：聚焦日本大学院申请场景，处理跨语言和跨院校信息匹配。
1

接入大模型 API

实现自然语言背景解析、个性化报告生成和更自然的咨询表达。

2

接入募集要项与官网信息

通过 PDF / 官网资料整理和 RAG 检索，提高申请资格、语言要求和招生信息的准确性。

3

完善用户侧工具

增加推荐报告导出、用户收藏、院校对比、导师对比和申请进度管理等功能。
